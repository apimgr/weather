# AI Behavior Rules (PART 0, 1)

⚠️ **These rules are NON-NEGOTIABLE. Violations are bugs.** ⚠️

## CRITICAL - NEVER DO

- Never guess or assume — always read the spec (AI.md) first
- Never say "done" without verifying the outcome
- Never run `git add`, `git commit`, or `git push` — write to `.git/COMMIT_MESS` instead
- Never add AI attribution anywhere (code, comments, commits, PRs, docs)
- Never add "Co-Authored-By: Claude/AI", "generated by AI", or similar
- Never add inline comments — comments ALWAYS go above the code
- Never implement a feature without reading the relevant PART in AI.md
- Never output "Found X issues" — fix them instead
- Never skip optional PARTs that are marked as implemented for this project
- Never modify PARTS 0-36 implementation patterns — they are fixed

## CRITICAL - ALWAYS DO

- Read AI.md relevant PART before implementing any feature
- Write to `.git/COMMIT_MESS` (never run git commands)
- Place all comments ABOVE the code they describe (never inline)
- Verify changes before claiming completion
- Use numbered/lettered options when asking user questions
- Update `TODO.AI.md` with newly discovered tasks
- Complete each feature FULLY before moving to the next
- Acknowledge when blocked — never silently skip

## Commit Message Rules

| Rule | Detail |
|------|--------|
| File location | `.git/COMMIT_MESS` (project root) |
| User command | `git commit -F .git/COMMIT_MESS` |
| ✅ emoji | MUST be used for todo completion commits |
| No changes | Do nothing — no COMMIT_MESS needed |
| Stale COMMIT_MESS | Recreate from scratch to match `git status` |
| No AI attribution | NEVER add Co-authored-by for AI tools |

## COMMIT_MESS Workflow

```
git status --porcelain → empty?       → Do nothing
                       → changes exist → Check COMMIT_MESS
                                        → Stale? Recreate
                                        → Missing? Create
                                        → Matches? Append new changes
```
